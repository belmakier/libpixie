1/* This file contains the details of every algorithm that could be used to process the traces.
2
3 You should edit/add to this file, the corresponding .cc file if you want to add your own algorithm.
4
5 Create a class that inherits from the PIXIE::Trace::Algorithm base class, and make sure you overload the Load, Process, and Prototype methods. Load should take a settings file and load appropriate settings into your object. Process is a method that actually processes the traces, outputting a vector PIXIE::Trace::Measurement objects - these must always be in the same order, and the method should always return the same number of them. Prototype() returns an equivalent vector, but with no trace needed. It is used only to determine how many measurements the particular algorithm returns, and get the names of the measurements for the Tree branch names.
6
7 If you're confused or don't understand, send me an email at <timothy.gray@anu.edu.au>
8*/
